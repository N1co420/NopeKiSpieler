ProjektPlanung und Dokumentation

Projektplan für Kartenspiel-Simulation

  1. Projektübersicht und Ziele:
    • Das Ziel des Projekts ist die Entwicklung einer Kartenspiel-Simulation, die es Benutzern ermöglicht, gegen KI-Spieler anzutreten oder an Turnieren teilzunehmen.
    • Das Hauptziel besteht darin, eine fähige KI zu implementieren.
  1. Funktionsanforderungen:
    • Implementierung eines Hauptmenüs für Benutzerinteraktionen und Spieloptionen.
    • Entwicklung der Spielregeln und der Logik für Kartenbewegungen und Spielzüge.
    • Integration von KI-Spieler mit eigenen Strategien.
    • Implementierung der Kommunikation mit einem Server für Turniermanagement und Spielerstatistiken.
  1. Dateistruktur und Modulplanung:
    • Dateien im "src"-Ordner:
      • main_menu.py: Hauptdatei für das Hauptmenü und die Benutzerinteraktion.
      • game_rules.py: Enthält die Spielregeln und die Logik für Kartenbewegungen.
      • ki_spieler.py: Implementierung von KI-Spielern mit verschiedenen Strategien.
      • payload_builder.py: Hilfsmodul zum Erstellen von Spielzügen und Payloads.
      • sio_events.py: Behandlung von Socket.IO-Ereignissen für die Serverkommunikation.
      • rest_api.py: Verbindung und Kommunikation mit dem Server über eine REST-API.
      • tournament_handler.py: Verwaltung von Turnieren, Spielrunden und Spielerstatistiken.
    • Dokumentation durch Markdown Files
  1. Funktionslogik und Pseudocode:

    a) main_menu.py

    • Funktion mainMenu():
      • Stellt eine Verbindung zum Server her und zeigt das Hauptmenü an.
      • Verarbeitet die Benutzereingabe und ruft die entsprechenden Funktionen auf, basierend auf der Auswahl.
      Funktion mainMenu():
    Verbinde mit dem Server
    Wiederhole, bis das Spiel beendet wird:
        Zeige das Hauptmenü an
        Verarbeite die Benutzereingabe:
            Wenn die Auswahl "q" ist:
                Erfrage die Anzahl der Spiele vom Benutzer
                Erstelle das Turnier mit der angegebenen Anzahl von Spielen
            Wenn die Auswahl "w" ist:
                Erfrage die Turnier-ID vom Benutzer
                Tritt dem Turnier mit der angegebenen ID bei
            Wenn die Auswahl "e" ist:
                Starte das aktuelle Turnier
                Wenn das Turnier erfolgreich gestartet wurde:
                    Zeige eine Erfolgsmeldung an
            Wenn die Auswahl "r" ist:
                Verlasse das aktuelle Turnier
            Wenn die Auswahl "t" ist:
                Trenne die Verbindung zum Server und beende das Spiel
            Sonst:
                Zeige eine Fehlermeldung an
